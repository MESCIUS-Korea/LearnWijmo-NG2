AngularJS 지시문

Wijmo 팀은 JavaScript 애플리케이션을위한 Google의 프레임 워크 인 AngularJS의 큰 팬입니다. AngularJS는 템플릿, 데이터 바인딩, MVVM, 웹 구성 요소 등을 제공합니다.

AngularJS의 주요 장점 중 하나는 MVVM 패턴을 지원한다는 것입니다. MVVM 패턴은 애플리케이션 로직이 모델 (JavaScript로 구현 된 컨트롤러라고도 함)에 포함되어 있고 화면은 뷰 (HTML)에 포함되어 있습니다.

이를 위해 AngularJS는 사용자 지정 HTML 요소 및 특성 인 지시문을 지원합니다. AngularJS에는 많은 수의 기본 제공 지시문이 포함되어 있으며 WinForms 또는 XAML 응용 프로그램에서 자체 컨트롤을 만들 수 있기 때문에 쉽게 구현할 수 있습니다.

Wijmo 5는 모든 제어에 대해 AngularJS 지시문을 제공합니다. 지시문은 wijmo.angular.js 파일에 정의되어 있으며 다음과 같은 코드를 작성할 수 있습니다: 

<div ng-app="app" ng-controller="appCtrl">
    
<p>This is a <b>FlexGrid</b> control:</p>
<wj-flex-grid items-source="data">
    <wj-flex-grid-column header="Country" binding="country"></wj-flex-grid-column>
    <wj-flex-grid-column header="Sales" binding="sales"></wj-flex-grid-column>
    <wj-flex-grid-column header="Expenses" binding="expenses"></wj-flex-grid-column>
    <wj-flex-grid-column header="Downloads" binding="downloads"></wj-flex-grid-column>
</wj-flex-grid>
</div>

AngularJS 응용 프로그램에서 Wijmo 5 사용

프로젝트에서 Wijmo 5 Angular 지시문을 사용하려면 응용 프로그램의 Wijmo 5 참조 항목에 설명 된대로 AngularJS, Wijmo 및 Wijmo의 AngularJS 지시문에 참조를 추가하십시오.

일단 참조 정보가 있으면, 다음과 같은 코드를 사용하여 여러분의 앱이 "wj"모듈에 의존하고 있다고 AngularJS에게 알려줍니다.: 

var app = angular.module('app', ['wj']);

응용 프로그램을 정의하면, 어떠한 AngularJS 응용 프로그램에서도 사용할 수 있는 데이터 및 로직를 제공하는 컨트롤러를 추가할 수 있습니다.

모든 Wijmo 5 지시어는 "wj"접두어로 시작하고 그 뒤에는 대시를 사용하는 컨트롤의 클래스 이름이옵니다. 지시문에는 동일한 규칙에 따라 컨트롤의 속성과 일치하는 특성이 있습니다.

일부 지시문은 중첩 된 하위 지시문을 지원합니다. 예를 들어, wj-flex-grid 지시어는 하나 이상의 wj-flex-grid-column 지시문을 포함 할 수 있으며 wj-flex-chart 지시문은 하나 이상의 wj-flex-chart-series 지시문을 포함 할 수 있습니다. 이로 인해 XAML과 매우 유사한 풍부하고 표현적인 마크 업 구문이 만들어집니다. MVVM의 진정한 이점을 얻으려면 이러한 유연성이 필수적입니다. 컨트롤의 모양과 레이아웃은 뷰에 의해 정의됩니다. 컨트롤러는 데이터 만 제공합니다.
