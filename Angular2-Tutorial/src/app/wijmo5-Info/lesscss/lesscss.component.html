LESS란 무엇인가?

LESS는 변수, 믹스 인, 연산 및 중첩 된 규칙을 추가하여 CSS를 확장하는 동적 스타일 시트 언어입니다.

  • 변수: 재사용 가능한 공통 값 (예 : 색상 및 스타일 정보).
  • Mixins: 새로운 스타일에서 사용하는 기존 스타일의 속성.
  • 작동: CSS 속성의 수학적 생성 및 조작.
  • 중첩 된 규칙: 다른 CSS 선택자 안에 CSS 선택기가 있습니다.

LESS는 확장 된 LESS 언어로 스타일 시트를 먼저 작성한 다음 일반 CSS로 컴파일 할 수 있기 때문에 CSS 전처리 기라고도합니다. LESS는 오픈소스이므로 클라이언트 측 또는 서버 측에서 실행하거나 컴파일하여 일반 CSS로 출력 할 수 있습니다.

다운로드, 자세한 문서 및 리소스는 다음 웹 사이트를 참조하십시오: http://lesscss.org.

LESS 사용하기

명령 줄에서 LESS를 사용하거나 브라우저 내 사용을 위해 JS 파일로 다운로드하거나 타사 응용 프로그램을 사용할 수 있습니다.

명령 줄

LESS 웹 사이트에서 특정 지침과 명령 줄 프롬프트 목록을 찾을 수 있습니다: 명령줄 사용방법.

In-browser

  1. less.js 사본을 다운로드하여 저장하십시오.
  2. 스타일 시트를 만들고 평소 .css 확장명 대신 .less 파일 확장명을 사용하여 저장하십시오.
  3. HTML 페이지의 <head> 요소에 다음 코드를 추가합니다: 
<link rel="stylesheet/less" href="styles.less">
<script src="less.js"></script>

참고 : 링크 요소에서 LESS가 브라우저에서 컴파일되도록하려면 rel = "stylesheet/less"로 일반적인 rel = "stylesheet"를 바꾸어야합니다. LESS 스타일 시트는 LESS 스크립트 앞에 와야합니다.

페이지가로드되면 less.js가 브라우저에서 LESS 코드를 처리하고 컴파일합니다. 이 방법은 LESS로 개발을 시작하는 편리한 방법이지만 성능이 중요한 프로덕션 환경에서는 권장되지 않습니다.

서드 파디 응용 프로그램

실제 환경에서는 LESS를 일반 CSS로 컴파일하는 것이 좋습니다. 명령 줄 프롬프트가 제공하는 것 이상을 원한다면 다양한 타사 도구를 사용할 수 있습니다. LESS 웹 사이트에는 다양한 플랫폼 및 IDE 용 컴파일러 및 편집자 목록이 있습니다. 온라인 레퍼런스 컴파일러를 참조하십시오. Online LESS Compilers.

변수 사용자 정의

우리는 LESS로 Wijmo 5 테마를 만들고 컴파일 된 CSS 사본과 함께 소스 파일을 사용할 수 있도록했습니다. 기존 테마를 업데이트하고 제공된 파일을 사용하여 새로운 테마를 만들 수 있습니다.

Wijmo 5 테마에는 wijmo.theme.ThemeName.css라는 파일 명명 구조가 있습니다. LESS를 사용하여 테마를 업데이트하려면 해당 wijmo.theme.ThemeName.less 파일을 찾아서 수정 한 다음 위에 나열된 방법 중 하나를 사용하여 다시 컴파일하십시오.

모든 테마는 기본 색상 및 스타일 옵션 세트를 기반으로합니다. 우리는 주제를 통해 재사용하는 공통 가치에 대한 변수를 선언합니다. LESS 변수는 @ 기호로 시작됩니다. 

@background: #f3f3f3;
@header: #54443b;
@primary: #2780ec;
@text: #26211f;
@button: #5f534c;
@tool-tip: #e5d9cf;
@grid-cell-border: true;
@grid-right-side-col: none;
@border-radius: 4px;
@background-grad: false;
@button-grad: false;
@header-grad: false;

기본 변수 외에도 import 문을 사용하여 LESS 파일을 추가로 포함하여 mixins로 사용합니다. 여기에는 색상 기능, 스타일 mixin 및 mixin guards가 포함됩니다.

@import "mixins/color-functions";
@import "mixins/guards";
@import "mixins/styles";
@import "mixins/chart";

컬러 함수는 선택된 초기 색상에 따라 더 넓은 색상의 팔레트를 생성하고 mixin guards는 대비를 보장하도록 설계되었습니다. mixin guards는 밝기에 따라 색상을 평가하고 이에 따라 CSS를 작성합니다. 영역의 배경색이 밝기가 50%보다 크면 해당 영역의 텍스트가 어두운 색상으로 렌더링되고 반대의 경우도 마찬가지입니다.

새로운 테마 만들기

기존 테마를 복제 한 다음 색상 및 스타일 옵션의 기본 세트를 수정하여 새 테마를 만들 수 있습니다. 다음은 색상 및 스타일 변수의 핵심입니다.

@background - 컨트롤의 배경색.
@header - 컨트롤 헤더의 배경색 (예 : FlexGrid, FlexChart)
@primary - 테마 전체에 사용되는 기본 액센트 색상
@text - 기본 텍스트 색상
@button - 버튼의 배경색.
@tool-tip - 툴팁의 배경색.
@border-radius - 모든 컨트롤의 전역 테두리 반경 설정
@grid-cell-border - FlexGrid 셀 테두리 토글. 모든 격자 테두리를 끄려면 'false'로 설정.
@grid-left-side-col - FlexGrid의 행 머리글 셀의 배경색(기본적으로 @header와 동일).
@background-grad - 컨트롤 배경 색상 그라디언트 토글. 그래디언트를 추가하려면 'true'로 설정
@button-grad - 버튼 배경 색상 그라디언트 토글. 그래디언트를 추가하려면 'true'로 설정.
@header-grad - 머리글 배경 색 그라디언트를 토글. 그래디언트를 추가하려면 'true'로 설정.

새 테마 배포

새 테마를 만들어 일반 CSS로 출력하면 스타일 디렉토리에 배치하고 프로젝트에 포함하십시오. 이제는 Wijmo 5 컨트롤이 포함 된 HTML 페이지에서 링크 할 수 있습니다.

<link href="styles/NewTheme.css" rel="stylesheet">
